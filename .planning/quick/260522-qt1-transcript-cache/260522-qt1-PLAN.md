# Quick Task 260522-qt1: Transcript Cache

## Goal

When the user re-uploads an updated chat export, voice messages that were already transcribed should
be resolved from a local cache — not sent to the Whisper worker again. New voice messages (those
not in cache) are transcribed normally.

## Tasks

### Task 1 — Create `assets/js/cache.js`

Create a localStorage-based transcript cache module.

**Files:** `assets/js/cache.js` (new)

**Action:**
- Export `getCachedTranscript(basename)` → string or null
- Export `setCachedTranscript(basename, text)` → void
- Storage key: `'voicefill_transcripts_v1'` — JSON object mapping basename → transcript text
- Errors are swallowed silently (localStorage unavailable, JSON parse errors, storage full)
- Only cache `'result'` status entries — `'error'` ([Audio unreadable]) is NOT cached

**Verify:** File exists, exports both functions

**Done:** `assets/js/cache.js` created

---

### Task 2 — Wire cache into `assets/js/ui.js`

Modify `dispatchTranscription` to read cache before sending to worker, and `onWorkerMessage`
to write cache when a transcription result arrives.

**Files:** `assets/js/ui.js`

**Action:**
1. Add import at top: `import { getCachedTranscript, setCachedTranscript } from './cache.js';`

2. In `dispatchTranscription(result)`:
   - After filtering `voiceMessages`, loop over them and separate cached vs uncached
   - For cached: call `updateRowInPlace({ status: 'result', filename, text: cached })` immediately
   - Set `transcribeTotal = voiceMessages.length` (all messages)
   - Set `transcribeDone = voiceMessages.length - uncached.length` (cached ones count as done)
   - If `uncached.length === 0`: call `updateSummaryLine` + `enableCopyDownload` and return early
   - Otherwise dispatch only `uncached` messages to the worker (same logic as before)

3. In `onWorkerMessage`, inside the `data.status === 'result'` branch (not 'error'):
   - Call `setCachedTranscript(data.filename, data.text)` before `updateRowInPlace`

**Verify:** On first upload, all voice messages are transcribed + cached.
On second upload of same/extended chat, cached messages resolve immediately without worker.

**Done:** Import added, `dispatchTranscription` modified, `onWorkerMessage` modified

## must_haves

### truths
- Cache uses `localStorage` — survives app restarts, no server needed
- Cache key is audio basename (e.g. `PTT-20240115-WA0001.opus`) — stable across re-exports of same chat
- `[Audio unreadable]` is NOT cached — transient decode errors shouldn't stick
- `[No speech detected]` IS cached — silence is a valid stable result

### artifacts
- `assets/js/cache.js` — new file
- `assets/js/ui.js` — modified (import + 2 call sites)
