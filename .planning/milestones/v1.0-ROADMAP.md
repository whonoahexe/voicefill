# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-05-22
**Phases:** 1–3
**Total Plans:** 7

## Overview

Three phases delivering a working Electron desktop app that transcribes WhatsApp (and Instagram) voice messages entirely offline. Phase 1 built the complete parse-to-output pipeline with placeholder transcripts. Phase 2 dropped in the real Whisper worker. Phase 3 packaged the finished app as an Electron binary and added Instagram support.

## Phases

**Goal**: A user can drop a WhatsApp export ZIP and receive a fully reconstructed chat log with voice message positions identified, errors annotated, and output ready to copy — using placeholder transcripts
**Depends on**: Nothing (first phase)
**Plans**: 3 plans

Plans:

- [x] 01-01: Walking Skeleton — project scaffold, JSZip vendor, drag-drop + file picker, ZIP extraction, thin DOM render (INPUT-01, INPUT-02, PARSE-01, PARSE-04, UI-01–04)
- [x] 01-02: Complete parser — full _chat.txt parse, voice detection, audio matching, error annotations, all input modes, without-media screen (INPUT-03, INPUT-04, PARSE-02, PARSE-03, ERR-01–04)

**Success Criteria (all met):**

### Phase 2: Whisper Worker

**Goal**: Voice message placeholders are replaced with real transcripts produced by Whisper running entirely inside a Web Worker — no API key, no server, nothing leaves the device
**Depends on**: Phase 1
**Plans**: 2 plans

Plans:

- [x] 02-01: Worker pipeline — audio decode+resample, RMS silence gate, dispatch queue, in-place row updates, summary line, model banner, button enable/disable (TRANS-01, TRANS-02, TRANS-03, TRANS-04, TRANS-05, ERR-02)
- [x] 02-02: CSS polish — transcript fade-in keyframe, pending/resolved/error states, model banner styling, disabled button states (TRANS-03, TRANS-04, TRANS-05)

**Success Criteria (all met):**
1. ✅ Progress bar shows model download progress with explicit "~40MB — downloads once, then cached" warning
2. ✅ "Transcribing X of Y voice messages..." updating per message without freezing
3. ✅ Voice messages produce accurate English transcripts inline as [Voice message: "..."]
4. ✅ Silent/near-silent audio skipped and annotated as [No speech detected]

**Completed:** 2026-05-21

**Key decisions:**
- RMS-only silence gate (no_speech_prob not exposed by transformers.js pipeline API)
- CDN import of @huggingface/transformers@4.2.0 via jsdelivr (vendoring deferred to Phase 3)
- Two-step OfflineAudioContext decode+resample (48kHz→16kHz)
- Pipeline singleton storing Promise (not resolved value) for single-download guarantee

## Milestone Summary

**Key Decisions:**

- Electron chosen as distribution target — eliminates file:// and HTTP serving problems
- WhatsApp "with media" export required in v1; "without media" shows friendly re-export instructions
- Parser handles both Android (PTT-*.opus) and iOS (00000023-AUDIO-*.opus) filename patterns from day one
- Whisper Worker uses onnx-community/whisper-tiny.en at dtype: 'q8' (~40MB); sequential queue
- JSZip loaded as classic script before ES module entry point — UMD build registers window.JSZip; load order critical
- innerHTML banned for all user-supplied content — textContent enforced throughout
- detectExportMode triple-condition guard prevents parse-only .txt from routing to without-media screen
- Two-step OfflineAudioContext: decode at 48kHz, resample to 16kHz — single-context approach does not auto-resample
- CDN import of @huggingface/transformers via jsdelivr — bare specifier fails in Electron renderer without bundler

**Issues Resolved:**

- WhatsApp "without media" detection bug (PARSE flag false positive) — triple-condition guard in detectExportMode
- Worker init-failure null guard — prevents crash if Worker construction fails
- Undeclared modelLoadMaxPct variable removed from worker.js
- parseInstagram JSZip missing try/catch for library load failure
- WASM eval-string blocked in packaged Electron renderer — fixed via CSP meta tag with unsafe-eval

**Issues Deferred:**

- Safari Ogg/Opus decoding unresolved — v1 targets Chrome/Edge only
- no_speech_prob gate: transformers.js pipeline API does not expose this value; RMS gate used instead
- ERR-02 ([Audio unreadable]) deferred to Phase 2 — cannot detect corrupt .opus without decoding
