    Write the new worker.js. Preserve the comment block from lines 1-16 of the stub (the message protocol documentation) verbatim at the top of the file. Everything after the comment block is the new implementation.

    The implementation has seven parts in this order:

    Part 1 — CDN import and env config (Pitfall 1 — bare specifier and node_modules paths fail in Electron renderer without a bundler):
    Import pipeline and env from the exact CDN URL: 'https://cdn.jsdelivr.net/npm/@huggingface/transformers@4.2.0'. Set env.allowLocalModels = false on the next line.

    Part 2 — Silence threshold constant:
    const RMS_SILENCE_THRESHOLD = 0.01 with comment "1% of max amplitude — standard silence gate (no_speech_prob not exposed by transformers.js pipeline API)".

    Part 3 — WhisperSingleton class:
    Class with static instance = null. Static async getInstance(progress_callback) method: if this.instance is null, set this.instance = pipeline('automatic-speech-recognition', 'onnx-community/whisper-tiny.en', { dtype: 'q8', progress_callback }). Return this.instance. Critical: stores the Promise object, not the resolved value. Awaiting the same Promise twice is safe and returns the same resolved pipeline — this is what prevents duplicate model downloads.

    Part 4 — Eager warm-up (D-01 — model download begins at Worker creation, before user submits a file):
    Call WhisperSingleton.getInstance with a progress_callback arrow function that calls self.postMessage(progressData). Chain .then(() => self.postMessage({ status: 'ready' })). This block runs at module load time. The 'ready' message fires exactly once per Worker lifetime — do not emit it anywhere else (Pitfall 6).

    Part 5 — Message handler self.addEventListener('message', async (e)):
    Guard: if e.data.type !== 'transcribe' return. Destructure audioData, filename, index, total from e.data. Wrap entire body in try/catch. In the catch block: console.warn('[VoiceFill Worker] Error processing', filename, err); self.postMessage({ status: 'error', filename, message: err.message }). No re-throw — the queue continues (ERR-02, D-11).
    Inside try:
    a. const transcriber = await WhisperSingleton.getInstance() — no callback, resolves from cached Promise.
    b. const samples = await decodeAndResample(audioData).
    c. RMS gate: if computeRMS(samples) < RMS_SILENCE_THRESHOLD, postMessage({ status: 'result', filename, text: '[No speech detected]', index, total }) and return. This is the complete silence gate — no_speech_prob is not available from the pipeline (TRANS-05, Pitfall 2).
    d. const result = await transcriber(samples) — bare Float32Array, no wrapper object.
    e. self.postMessage({ status: 'result', filename, text: result.text.trim(), index, total }).

    Part 6 — computeRMS(float32Array):
    Function that sums float32Array[i] * float32Array[i] in a for loop, then returns Math.sqrt(sum / float32Array.length). Returns 0 for an empty array (handle: if float32Array.length === 0 return 0).

    Part 7 — async decodeAndResample(uint8Array):
    Two-step decode and resample. Step 1: const decodeCtx = new OfflineAudioContext(1, 1, 48000); const decoded = await decodeCtx.decodeAudioData(uint8Array.buffer.slice(0)). The .slice(0) is mandatory — decodeAudioData detaches the ArrayBuffer, and a Uint8Array from JSZip or File.arrayBuffer() may be a view into a larger shared buffer (Pitfall 3). The context sampleRate of 48000 does not affect the decoded audio's sample rate — decodeAudioData output rate matches the source file (WhatsApp Opus is 48kHz). Step 2: const targetRate = 16000; const targetLength = Math.ceil(decoded.duration * targetRate); const resampleCtx = new OfflineAudioContext(1, targetLength, targetRate); const source = resampleCtx.createBufferSource(); source.buffer = decoded; source.connect(resampleCtx.destination); source.start(0); const resampled = await resampleCtx.startRendering(); return resampled.getChannelData(0). This Float32Array at 16kHz is what pipeline() receives.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Worker postMessage -> main thread | Worker-returned transcript text arrives via postMessage; if tampered, could contain malicious strings |
| User audio file -> Worker | .opus bytes from user's ZIP/folder are processed by OfflineAudioContext; malformed files could cause decode errors |
| CDN -> Worker module import | transformers.js library fetched from CDN at first startup; network interception could deliver tampered code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | updateRowInPlace() transcript text | mitigate | textContent-only assignment — enforced unconditionally; no innerHTML for Worker-returned strings (XSS rule T-03-01 from Phase 1) |
| T-02-02 | Tampering | updateBanner() / updateSummaryLine() | mitigate | textContent-only for all dynamic text including numeric progress values — same XSS rule applied uniformly |
| T-02-03 | Tampering | CDN import of @huggingface/transformers | mitigate | Version pinned to exact release @4.2.0 in CDN URL; CDN HTTPS provides transport security; Phase 3 will vendor the file |
| T-02-04 | Denial of Service | OfflineAudioContext.decodeAudioData() on corrupt Opus | accept | Throws DOMException caught by try/catch in worker.js; Worker emits error and continues queue — no crash, no DoS |
| T-02-05 | Elevation of Privilege | Malicious .opus triggering Chromium audio decoder exploit | accept | Runs inside Worker sandbox + Chromium renderer sandbox; low-value target (local tool, no sensitive data); Chromium audio decoder is maintained by Google |
| T-02-SC | Tampering | No npm/pip/cargo installs in this plan | accept | No new packages installed; CDN import is the only external fetch; covered by T-02-03 |
</threat_model>
