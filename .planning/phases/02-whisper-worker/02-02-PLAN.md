  <action>
    Add four rule groups to style.css. Do not remove or modify any existing rules. Insert each group near its semantically related existing rules:

    Rule Group 1 — Transcript fade-in keyframe and .resolved state (D-06):
    Place adjacent to or after the @keyframes dot-blink rule.
    Keyframe name: transcript-appear. From: opacity 0. To: opacity 1. Duration: 350ms, ease-in, forwards fill mode. This duration matches the "ink appearing on parchment" aesthetic described in CONTEXT.md §Specific Ideas (300-500ms range, subtle).

    .voice-annotation.resolved rule:
      animation: transcript-appear 350ms ease-in forwards
      opacity: 1  (explicit final state for browsers that don't respect forwards fill mode)
      font-style: normal  (resolved transcripts are roman, not italic — visual distinction from pending state)

    Rule Group 2 — .pending state (D-06 complement):
    Place after or near the .voice-annotation.resolved rule.
    .voice-annotation.pending rule:
      font-style: italic
      opacity: 0.6

    The .error state for voice rows is already handled by the existing .voice-annotation.error class if it exists, or inherits from .voice-annotation base — check existing styles. If no .error rule exists, add: .voice-annotation.error { font-style: italic; color: var(--color-accent); } to distinguish unreadable audio rows.

    Rule Group 3 — Model download banner (D-07 — "footnote, not a status bar"):
    Place near the #screen-upload rules block.
    #model-banner rule:
      display: none  (hidden by default; JS sets display:block on progress events, then display:none on ready)
      font-size: 12px
      opacity: 0.6
      margin-top: 16px
      font-style: italic
      text-align: center
    No border, no background color, no padding — it must feel like a footnote, not a notification widget. The parchment background shows through.

    Rule Group 4 — Disabled button states (D-05):
    Place after the .btn-primary:active rule (or wherever :active states are defined).
    .btn-primary:disabled rule:
      opacity: 0.4
      cursor: not-allowed
    .btn-secondary:disabled rule:
      opacity: 0.4
      cursor: not-allowed
    These complement the JS opacity setting already applied in enableCopyDownload/disableCopyDownload — the CSS :disabled selector provides the cursor change and serves as a declarative fallback.

    Section comment convention: prefix each new rule group with a section comment in the same style as existing comments (e.g., "/* Phase 2 — Transcript animation */").
  </action>
